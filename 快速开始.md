# 快速开始

校园干洗店管理系统 - 本地运行指南

## 环境要求

| 依赖 | 版本 | 说明 |
|------|------|------|
| **.NET SDK** | **10.0 或以上** | 下载：https://dotnet.microsoft.com/download/dotnet/10.0 |
| **SQL Server LocalDB** | SQL Server 2022 Express LocalDB 或更高 | 随 Visual Studio（任意工作负载）自动安装。如未安装，可单独下载：https://go.microsoft.com/fwlink/?linkid=2216919 |
| **操作系统** | Windows 10 或以上 | 本项目使用 Windows 身份验证连接数据库，仅支持 Windows |

## 快速运行

```bash
# 1. 克隆项目
git clone https://github.com/Azz-516/xiaoliran.git
cd xiaoliran

# 2. 还原依赖
dotnet restore

# 3. 运行
dotnet run
```

运行成功后，浏览器访问：
- HTTP：http://localhost:5079
- HTTPS：https://localhost:7181

## 数据库说明

### 连接配置

数据库连接字符串位于 **`Program.cs` 第 33 行**：

```csharp
options.UseSqlServer("Server=(localdb)\\MSSQLLocalDB;Database=cleandb;Trusted_Connection=True;TrustServerCertificate=True;");
```

- 使用 **LocalDB**（轻量级 SQL Server，无需额外安装）
- 使用 **Windows 身份验证**（`Trusted_Connection=True`），无需配置用户名/密码
- 数据库名称：`cleandb`

### 如需切换为完整 SQL Server

将 `Program.cs` 第 33 行的连接字符串修改为：

```csharp
options.UseSqlServer("Server=localhost;Database=cleandb;User Id=sa;Password=你的密码;TrustServerCertificate=True;");
```

### 数据库初始化

**无需手动建表或导入数据**，程序每次启动时会自动执行以下操作：

1. **`db.Database.EnsureCreated()`** — 如果 `cleandb` 不存在则自动创建，并自动创建全部 7 张表
2. **`SeedInitialData(db)`** — 初始化基础数据（仅首次运行生效）
3. **`SeedTestData(db)`** — 注入测试数据（仅首次运行生效）

### 初始化内容

| 数据 | 数量 | 说明 |
|------|------|------|
| 角色 | 2 | 管理员（admin）、普通用户（user） |
| 权限 | 5 | 查看首页、用户管理、门店管理、订单管理、查看我的订单 |
| 角色-权限关联 | 6 | admin 拥有全部 4 项权限，user 拥有 2 项权限 |
| 门店 | 54 | 初始 4 家 + 测试 50 家 |
| 测试用户 | 50 | 用户名格式 `test_user_001` ~ `test_user_050`，密码均为 `123456` |

### 重置数据库

如需清除所有数据并重新开始：

```bash
# 方式一：删除数据库（推荐）
sqlcmd -S "(localdb)\MSSQLLocalDB" -Q "IF DB_ID('cleandb') IS NOT NULL BEGIN ALTER DATABASE cleandb SET SINGLE_USER WITH ROLLBACK IMMEDIATE; DROP DATABASE cleandb; END"

# 然后重新启动
dotnet run
```

## 常见问题

**Q: 启动时报 "No database provider has been configured" 错误**
A: 确保已安装 .NET 10.0 SDK，运行 `dotnet --version` 检查。

**Q: 启动时报 "A network-related or instance-specific error" 错误**
A: LocalDB 未安装或未启动。安装 Visual Studio 任意工作负载即可自带 LocalDB。

**Q: 启动后页面显示 500 错误**
A: 检查 `logs/log-*.txt` 日志文件查看详细错误信息。

**Q: 数据库文件存储在哪里**
A: LocalDB 的数据库文件默认存储在 `%USERPROFILE%` 目录下（SQL Server 自动管理）。
