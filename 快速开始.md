# 校园干洗店管理系统 — 快速开始

## 项目简介

基于 ASP.NET Core 10.0 Razor Pages 的校园干洗店管理系统，支持管理员与用户双角色。管理员可管理用户、门店和订单；用户可浏览门店、在线下单和查看个人订单。

**技术栈**：ASP.NET Core 10.0 · EF Core 10.0 · SQL Server · Bootstrap 5 · jQuery · Serilog

## 2. 配置数据库连接

打开 **`appsettings.json`**，修改第 3 行的连接字符串：

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=(localdb)\\MSSQLLocalDB;Database=cleandb;User Id=sa;Password=xiaoliran!;TrustServerCertificate=True;"
  }
}
```

### 常见连接方式

**方式一：Windows LocalDB（推荐用于开发）**
```
Server=(localdb)\\MSSQLLocalDB;Database=cleandb;User Id=sa;Password=你的密码;TrustServerCertificate=True;
```

**方式二：本地 SQL Server 实例**
```
Server=localhost;Database=cleandb;User Id=sa;Password=你的密码;TrustServerCertificate=True;
```

**方式三：远程 SQL Server**
```
Server=192.168.1.100;Database=cleandb;User Id=sa;Password=你的密码;TrustServerCertificate=True;
```

> **注意：** LocalDB 需要 SQL Server 用户（sa）已配置密码。如未启用 SQL 身份验证，请在 SQL Server Management Studio 中为 sa 用户设置密码并启用混合模式认证。

## 4. 数据库初始化（全自动，无需手动操作）

项目启动时会自动完成以下操作：

### 第一步：自动建库建表
- 调用 `db.Database.EnsureCreated()`
- 如果连接字符串中指定的数据库（如 `cleandb`）**不存在**，则自动创建
- 自动创建全部 7 张表：`tb_user`、`tb_role`、`tb_user_role`、`tb_permission`、`tb_role_permission`、`tb_laundry_shop`、`tb_order`
- **无需手动建表、无需执行 SQL 脚本、无需 `dotnet ef database update`**

### 第二步：初始基础数据（仅首次生效）
| 数据类型 | 数量 | 说明 |
|----------|------|------|
| 角色 | 2 | 管理员（admin）、普通用户（user） |
| 权限 | 5 | 查看首页、用户管理、门店管理、订单管理、查看我的订单 |
| 角色-权限关联 | 6 | admin 拥有 4 项权限，user 拥有 2 项权限 |
| 门店 | 4 | 东校区、西校区、南校区、北校区各一家 |
| 管理员 | 1 | 用户名/密码：`xiaoliran`，姓名：李然，角色：管理员 |

### 第三步：注入测试数据（仅首次生效）
| 数据类型 | 数量 | 说明 |
|----------|------|------|
| 测试用户 | 50 | 用户名 `test_user_001` ~ `test_user_050`，密码均为 `123456`，已分配普通用户角色 |
| 测试门店 | 50 | 随机生成的门店名称和地址 |

> **安全机制：** 每一步都有存在性检查，重复启动不会重复插入数据。

### 如何重置数据库

如需清除所有数据重新开始：

```bash
# 方式一：通过 sqlcmd 删除数据库
sqlcmd -S "(localdb)\MSSQLLocalDB" -U sa -P "xiaoliran!" -Q "ALTER DATABASE cleandb SET SINGLE_USER WITH ROLLBACK IMMEDIATE; DROP DATABASE cleandb;"

# 方式二：通过 SQL Server Management Studio 右键删除数据库

# 然后重新启动，数据库会自动重建
dotnet run
```

## 6. 功能概览

## 8. 常见问题

**Q: 启动时报 "A network-related or instance-specific error"**
A: SQL Server 连接失败。检查 `appsettings.json` 中的连接字符串是否正确，数据库服务是否已启动。

**Q: 启动时报 "Cannot open database 'cleandb' requested by the login"**
A: 数据库不存在且无法自动创建。可能是 sa 用户权限不足，请手动创建空数据库 `CREATE DATABASE cleandb` 后重新启动。

**Q: 启动后页面显示 500 错误**
A: 检查 `logs/log-YYYYMMDD.txt` 日志文件查看详细错误信息。

**Q: 登录后报错或权限不对**
A: 检查数据库中 `tb_role` 和 `tb_permission` 表是否有初始数据。如果没有，删除数据库后重新 `dotnet run`。

**Q: 如何修改端口**
A: 编辑 `Properties/launchSettings.json` 中的 `applicationUrl`。

## 10. 技术说明

| 项目 | 说明 |
|------|------|
| 密码存储 | **明文存储**（无哈希），仅用于演示 |
| 身份认证 | Session 会话 + 基于角色的访问控制（RBAC） |
| 数据迁移 | 使用 `EnsureCreated()`，不使用 EF Migrations |
| 表命名 | 所有表统一使用 `tb_` 前缀 |
| 时间字段 | `CreateTime` 使用 `datetime` 类型，默认 `GETDATE()` |
| 分页参数 | 使用 `?p=N` 而非 `?page=N`（避免与 Razor Pages 路由冲突） |
| 防伪标记 | 管理页面使用 `[IgnoreAntiforgeryToken]` 属性 |
| 日志 | Serilog，最小级别 Warning，输出到控制台 + `logs/` 目录 |
